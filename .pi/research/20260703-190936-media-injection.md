## 2. What transformations it performs

### 2.1 Pipeline: `media_library.py` (6-stage Click CLI)

```
inventory → classify → convert → finalize → qa → [signoff → archive]
                                               ↑ optional halt gate
```

| Stage | Input → Output | Key logic |
|-------|---------------|-----------|
| **inventory** | Raw files → `manifest.json` with per-entry metadata | Scans `templates/media/` (skipping `reference/`, `_staging/`, `_raw_archive/`). Parses SVG viewBox + text count. Computes pHash for rasters. Detects file openability. Detects family groups by filename prefix pattern. Injects Envato metadata for `_envato_ingest/` files via `_inject_envato_meta()`. |
| **classify** | Manifest entries → updated `manifest.json` + `classification-review.md` | Keyword-based regex rules (20 rules) against normalized filename. SVG-type filter for backgrounds (≥3900×≥2200 viewBox, zero text → `background` at 1.0 confidence). Pattern-based rules (e.g. `\d+[ -]?step` → `process`). Falls back to `uncategorized` (0.4 confidence). Flags for review if: uncategorized, or top-2 candidates within 0.2 confidence, or confidence <0.7, or non-representative in a family group. **Preserves Envato-injected classifications** (skips re-classification when `category_source == "envato"`). |
| **convert** | Raw files → `_staging/*.png` | SVG: renders at max 1920px longest side via `resvg-py` (primary) or `cairosvg` (fallback). Raster: opens with PIL and re-saves as PNG. Re-computes pHash on the staged PNG. Tracks failures per-entry. |
| **finalize** | Staged PNGs → `reference/library/{category}/{category}-NNN.png` | Copies staged PNGs into category subdirectories with stable sequential names. Writes README.md per category with source-style, structural-elements, reusable-for-BAMi, and ignore annotations. Writes master index at `library/README.md`. |
| **qa** | Library artifacts → `_qa/qa-report.md`, `_qa/coverage.md`, `_qa/duplicates.json` | Reconciliation (converted vs failed). Low-resolution flagging. README coverage check (every entry has a README row). Classification review gate listing. pHash near-duplicate detection (Hamming distance ≤5). Coverage summary with status per category. Sets `qa_ready` boolean. |
| **archive** | Originals → `_raw_archive/` | Moves successfully converted originals out of the raw root. Gated by `qa_signoff` (or `--force`). |
| **signoff** | QA report → QA approval recorded | Records `qa_signoff: true` in manifest. Requires qa report to be fresher than manifest. |
| **restore** | `_raw_archive/` → raw root | Rollback helper. |
| **full** | Runs all 6 stages sequentially | Optional `--force-archive` to bypass signoff gate. |

### 2.2 Pipeline: `tools.envato_assets` (7-stage Click CLI)

```
inventory → extract → calibrate → classify → catalog → handoff
         ↓ optional stop
      [excluded]              calibrate (optional sample-based tuning)
                                       ↓
                                 threshold gate
                              (halt if >15% flagged)
```

| Stage | Input → Output | Key logic |
|-------|---------------|-----------|
| **inventory** | Envato ZIPs → `_processing_state.json` + excluded report | Scans each ZIP. Detects layout type (A–H). Checks for processable vector files (.ai, .pdf, .svg). Excludes FIG/XD/AF/PSD-only packs. Records per-pack metadata (vector_count, member_count, Nested_zips). |
| **extract** | Vector files → crop PNGs in `_extract_cache/` + `_envato_ingest/` | Opens AI/PDF via PyMuPDF. Plans crops per page using OpenCV contour detection (`plan_crops`). Renders each crop to PNG. Copies to `_envato_ingest/` with stable name. Stores per-crop metadata in `_crop_index.json` (strategy, dimensions, review flags). |
| **calibrate** | Fixed sample of 6 diverse packs → review rate evaluation | Runs extraction + classification on a fixed sample. Evaluates review-flagged rate. Halts if >15% flagged. Purges stale calibration rows before re-run. |
| **classify** | Crop index entries → enriched crop index | Hybrid: seed from discovery taxonomy (11-category → 20-category mapping via `SEED_TO_LIBRARY_MAP`), then keyword refinement (20 rules on filename + text blocks). Computes rich metadata: slot_count (heuristic from colour-region edge density), orientation, text_capacity, color_style. Optional vision endpoint (`BAMI_VISION_ENDPOINT` env var) for ambiguous cases. |
| **catalog** | Crop index → CSV/JSON projections + contact sheet | Writes `_asset_catalog.csv` (18 fields), `_asset_catalog.json`, `_processing_report.md`, and a deterministic 10% sample contact sheet PNG. |
| **handoff** | Envato crop index → media_library pipeline | Sets `media_library._ENVATO_CROP_INDEX_OVERRIDE`, then calls `media_library.inventory/classify/convert/finalize/qa` on the combined corpus. Clears override afterward. |
| **full** | Orchestrates all of the above | Runs inventory → extract → classify → catalog → handoff with stop-condition gates. |

### 2.3 Supported formats

| Input format | Processing method | Output format |
|-------------|-------------------|---------------|
| `.webp` | PIL open + re-save as PNG | `.png` |
| `.png` | PIL open + re-save as PNG | `.png` |
| `.jpg` / `.jpeg` | PIL open + re-save as PNG | `.png` |
| `.svg` | `resvg-py` (primary) or `cairosvg` (fallback) at 1920px longest edge | `.png` |
| `.ai` | PyMuPDF (fitz) → page raster → crop extraction | `.png` |
| `.pdf` | PyMuPDF (fitz) → page raster → crop extraction | `.png` |

## 4. How QA/signoff/archive works

### 4.1 QA gates

| Gate | Trigger | Blocking? | Bypass? |
|------|---------|-----------|---------|
| **Classification review** | `classify` step generates `classification-review.md` | Non-blocking (informational) | N/A |
| **QA report** | `qa` step generates `qa-report.md` | Non-blocking (informational) | N/A |
| **QA signoff** | Human reviews QA report, runs `signoff` command | Blocks `archive` | `archive --force` |
| **Envato calibration review rate** | `calibrate` step evaluates >15% threshold | Halts `full` pipeline | `--skip-review-gate` |
| **QA step in handoff** | Envato → media_library handoff | Non-blocking in pipeline | `--skip-qa` |

### 4.2 Artifacts produced for human review

1. **`classification-review.md`** — Lists every review-flagged entry with its chosen category, candidate categories with confidence scores, and group-representative status. Also lists decorative background SVGs and all family groups.
2. **`qa-report.md`** — Full reconciliation: raw count, converted count, failures, low-res flags, README orphans, review-flagged items, near-duplicate pairs, category coverage.
3. **`coverage.md`** — Per-category summary with status: `ok`, `needs more examples` (1 file), `low-res warning`, `empty`.
4. **`duplicates.json`** — pHash near-duplicate pairs within Hamming distance ≤5.
5. **`_processing_report.md`** (Envato) — Pack-level summary, per-category breakdown, review rate.
6. **`_qa_contact_sheet.png`** (Envato) — Deterministic 10% sample montage for visual spot-check.

## 5. How this supports slide/layout design and pattern injection

### 5.1 Architectural role

The media library is the **visual reference corpus** for the presentation framework's layout engine. It serves two distinct consumption paths:

**Path A — Semantic layout key mapping (benchmarks)**

The 2 hand-curated benchmarks (`reference-comparison-panel.png`, `reference-gantt-matrix.png`) are 1:1 mapped to semantic layout keys (`comparison_panel`, `gantt`). These are the *expected output* that layout generators should match structurally.

**Path B — Pattern injection / reference reuse (library)**

The 76 auto-categorized library PNGs are the "inspiration corpus" — visual examples of what each layout category looks like, drawn from real marketplace templates. When the presentation engine needs to decide *what structure* to use for a given content type, it can reference the library entries for that category.

### 5.2 Per-category annotations

Each category README provides 5 columns for every asset:

| Column | Meaning | Example |
|--------|---------|---------|
| **Source style** | What the source looks like | `"template-store slide thumbnail with marketplace styling; use only composition and information hierarchy"` |
| **Structural elements** | What visual structure defines this category | `"nodes, connectors, hub-spoke branches, directional relationships"` (for `flow`) |
| **Reusable for BAMi** | Whether the pattern can be reused directly | `"yes"` / `"no"` / `"partial"` |
| **Ignore** | What to discard from the source | `"color palette, decorative icons, font choices, non-BAMi chrome"` |

This is a **structural-only** reuse model — the library preserves *information hierarchy and composition*, explicitly discarding visual chrome (colors, fonts, icons).

## 7. Limitations still relevant to docs

### 7.1 Classification accuracy

- **30 of 76 entries are review-flagged (39.5%).** Most are either SVG icons that fall through to `uncategorized` (6), or the `7747-01-animated-powerpoint-charts-collection` family (10 entries from a single template pack that are all uncategorizable from filenames alone).
- The keyword-based classifier has no visual analysis for the legacy corpus — it relies entirely on filenames. An asset named `21601-01-marketing-pie-chart-powerpoint-template-16x9-1.webp` hits no keyword rule for "chart" or "pie" and lands in `uncategorized`.
- The `infographic-element` category has 0 assets because the `infographic` keyword rule has confidence 0.75, and any higher-confidence rule (e.g. `process` at 0.95) wins first.

### 7.2 SVG handling

- **resvg-py is required** for SVG rasterization. If it fails to import, fallback is `cairosvg` which requires a native Cairo runtime. If neither works, all SVG conversions halt with a runtime error.
- SVG classification is a single heuristic (large viewBox + zero text = `background`). Small SVGs with text (e.g. `12978999_5095296.svg` at 500×500 with 11 text nodes) fall through to `uncategorized`.

### 7.3 Envato extraction robustness

- **OpenCV contour-based crop extraction** (`plan_crops` in `cluster.py`) may produce false positives or miss crops depending on vector file structure. The 15% review-flagged threshold in calibration exists because of this.
- **AI/PDF files** processed via PyMuPDF may have unexpected page structures — the code reads all pages and attempts cropping on each.
- **Nested ZIP depth** limited to 2.
- **The `_RECURSIVE_LOCK`** (a thread-level lock in `cluster.py`) gates document opening to prevent concurrent PDF access issues.

### 7.4 Bridge coupling

- The `_ENVATO_CROP_INDEX_OVERRIDE` module-level list in `media_library.py` is a **temporary in-memory bridge** set by the Envato pipeline before calling `inventory()`. It is cleared in a `finally` block in `handoff()`, but if `handoff()` is interrupted (exception, crash), the override persists in the module, potentially corrupting a subsequent standalone `media_library.py` run.
- **No persistent cross-reference** exists between `_crop_index.json` (Envato) and `manifest.json` (media library). The Envato metadata is injected transiently during inventory and lost once the manifest is written.

### 7.5 QA signoff gap

- The pipeline produces a `qa_ready: true` recommendation (0 failures), but **qa_signoff is false**. No human has reviewed the 30 flagged items or approved the categorization. The archive is gated behind signoff.

### 7.6 Missing test infrastructure

- `configure()` takes a `media_root` parameter for test isolation, suggesting tests exist, but the test files are not in the visible tree. The `--force` skip exists for manual testing bypasses.

### 7.7 Vision endpoint (optional)

- `BAMI_VISION_ENDPOINT` env var enables an optional vision-based reclassification for ambiguous crops. No endpoint was configured during this run. This is the primary path to improving the 39.5% flagged rate for the Envato crops, but it adds a network dependency and API cost.
