# Implementation Summary — Radically expand BAMi slide templates, layouts & content primitives

## Scope implemented

### Phase A — Foundations & de-risking
| Task | Status | Description |
|------|--------|-------------|
| **1. Body-zone single source** | ✅ | Removed hardcoded `_BODY_TOP`/`_BODY_BOTTOM` from `build.py` (was 1.0/10.5) and `blocks.py` (was 1.2/10.5). Both now read from `design_tokens.yaml` `grid.body_zone`. Added `_body_zone_from_tokens()` helper and per-template `body_zone` override support. |
| **2. Schema single source + versioning** | ✅ | `shared/pptx/schema.py` now loads `schemas/content-schema.json` at import time instead of duplicating the inline dict. Added `schema_version` (starts at 2) with migration from legacy decks. JSON updated with new fields: `schema_version`, `layout`, `variant`, `content`, `section_divider` template, new block kinds. |
| **3. Data-driven dispatch** | ✅ | Replaced `if tname == "content"` branching with capability lookup from `design_tokens.yaml` (per-template `capabilities: {has_body, has_blocks, body_clears}`). Cover/closing are `{false, false, false}`; content is `{true, true, true}`. `section_divider` placeholder ready. |
| **4. Archetype hints in validator** | ✅ | Generator writes `BAMI::template=...` to each slide's notes. Validator reads notes first via `_read_archetype_hint()` before falling back to logo-position heuristics. Both per-slide chrome assertions and structure checks use hints. |
| **5. Constraints** | ⏳ | Schema updated with field kinds. Full `constraints.py` and pre-render overflow handling deferred to dedicated layout/constraint pass when the layout registry (Phase C) is wired. |
